3次握手

第一次握手：client发送SYN=1到server,设seq=J，client变成了SYN_SEND
第二次握手：server收到后，发送ACK=1,SYN=1，并发送ack=J+1,seq=M,server变成SYN_RECEIVED
第三次握手：client收到后回复ACK=1，ack=M+1，client和server变成established

4次挥手

第一次挥手：client发送FIN M到server,client变成FIN_wait
第二次挥手：server发送ack =M+1到client，server变成CLOSE_wait
第三次挥手：server发送FIN N到client，server编程LAST_ACK
第四次挥手：client发送ACK=1，ack=N+1到server,client进入TIME_WAIT,server收到了ack=N+1后进入CLOSED

七层/五层/TCPIP协议

七层：物理层，数据链路层，网络层，传输层，会话层，表示层，应用层
五层：物理层，数据链路层，网络层，传输层，应用层
TCP/IP:网络接口层，网络层，传输层，应用层

ARP协议

ARP协议是地址解析协议，从IP转到MAC地址。
ARP缓存中存有ARP列表，是一个IP到MAC的映射表，有数据需要发送时，先查看ARP列表中是否有目的的MAC地址，如果有则发送，没有进行广播，将源IP-MAC和目的的IP打包广播在网络中。
网络中的节点判断这个目的是不是自己的，同时将源的IP-MAC写入自己的ARP表中，如果已有则覆盖，同时将自己的IP-MAC传回去。
如果主机一直未收到，则查询失败。

TCP和UDP的区别

  1. TCP是面向连接的服务，即通信前要建立一条长时间保持的通道，是一种比较可靠的数据传输。
    UDP是非面向连接的，相对不可靠，但是更块。
  2. TCP传输的是报文段，UDP传输的是用户数据。
  3. TCP对应的有FTP,HTTP,SMTP,POP3;UDP对应的DNS。

DNS作用

DNS是将域名转换为IP地址，不直接为用户工作，而是服务与其他的协议。使用UDP协议。
每次发送三条信息，DNS类型，查询的域名，域名的类别。端口号53。

浏览器输入baidu.com的过程

  1. 在应用层，通过DNS解析网站的IP地址。客户端发起一个HTTP请求到IP地址。
  2. 在传输层，TCP进行包装成报文段，加入源和目标的端口号。
  3. TCP包传到网络层，通过路由找到一条通往IP地址的最优路径。
  4. 包通过链路层，然后发送ARP请求查找目的地址，如果得到回应，发送IP数据包到达服务器的地址。

HTTP和HTTPS的区别

  1. HTTP 的 URL 以 http:// 开头，而 HTTPS 的 URL 以 https:// 开头
  2. HTTP 是不安全的，而 HTTPS 是安全的
  3. HTTP 标准端口是 80 ，而 HTTPS 的标准端口是 443
  4. 在 OSI 网络模型中，HTTP 工作于应用层，而 HTTPS 工作在传输层
  5. HTTP 无需加密，而 HTTPS 对传输的数据进行加密
  6. HTTP 无需证书，而 HTTPS 需要认证证书
