面试题之“C++基础”

发表于 2016-12-10   |   分类于 面试题目   |  

语言基础

  1. 关键词含义
    const:对象在程序执行期间不能被修改
    static:
    • 局部变量：函数调用期间保持值
    • 类变量：被所有的成员共享。如果是类的成员函数，用类：：函数名访问，只能访问静态数据。
    • 全局变量：作用范围是文件内
      extern:其他文件中可以得到已定义变量的引用
      mutable：可以通过成员函数修改
  1. i+ + 和 + +i:
    前缀形式：表达式计算前完成自增或自减
    后缀形式：表达式计算后完成自增自减
  2. x<<1 左移一位，相当于乘2="" x="">>1 右移一位，相当于除2
  3. 函数的声明中，参数名不重要，参数类型重要
  4. 数组作为函数的参数，会退化为一个指向数组的指针

面向对象

  1. 类成员函数

    • 在类内定义，默认是内联的
    • 在类外定义，使用范围解析运算符：：

  2. 类访问修饰符

    • public：外部可以访问
    • private:只有类成员函数和友元可以访问
    • protect：在子类中可以访问

  3. 构造和析构

    • 构造：和类名相同，没有返回值。可以使用初始化列表来初始化值。
    • 析构：和构造相对，~

  4. 拷贝构造函数
    classname(const classname &obj):{}
    复制对象；将对象作为函数参数

  5. 友元函数
    定义在类外，声明在类内。可以访问类的私有和公有成员。

  6. 内联函数
    相当于宏。在编译时，编译器会把该函数的代码副本放在每个调用它的地方。
    内联函数必须和函数定义在一起，而不是声明。

  7. this指针
    this是所有成员函数的隐含参数，this->变量

  8. 指向类的指针
    用类的指针访问成员变量和函数，必须用->

  9. 继承

    • C++可以多继承
    • 继承类型 Public，protect,private,继承使得子类中的访问类型，最高为继承类型。

  10. 多态
    使用指针指向子类方法时，会调用基类方法，属于静态绑定。
    基类方法加上vitrual，会指向子类方法，变成动态绑定。

  11. 和结构体的区别
    结构体也有成员函数和继承，主要的区别在于，结构体默认的访问控制是public，类是private。

const

  • const 和 #define区别

    1. const有数据类型，宏定义没有
    2. 编译器会对const进行类型安全检查，而宏没有
    3. 有些调试工具会对const进行调试，但不能对宏调试

  • 类中的常量

    1. 只能在构造函数的初始化列表中
    2. 采用枚举型常量

  • const修饰成员函数

    1. 不能调用其他非const函数
    2. 不能修改数据成员
    3. const修饰成员函数后，mutable的成员变量可以修改

  • 常量指针和指针常量

    1. 常量指针： const int *p（从左往右念，先常量后指针）相当于一个敞亮的指针，即指针指向的值不能变，但是可以更改指向。
    2. 指针常量： int * const p，即不能改变指向的指针。

指针

  • 引用和指针的区别

    1. 引用在创建时必须初始化
    2. 不能用NULl引用
    3. 引用不能修改引用的关系

  • 指针和数组的比较：

    1. 数组在静态存储区或者栈上，数组名对应一块内存，地址和容量保持不变。
    2. 指针指向任意类型的内存，常用指针操作动态内存。

  • 指针使用的注意点：

    1. 指针指向一个常量字符串，不能修改
    2. 不能对数组名进行比较和复制
    3. 计算内存容量是，sizeof(p)返回的是4，得到一个指针变量的字节数，相对应sizeof(char[])返回的数组所占的字节数。数组作为函数参数进行传递时，会退化为同类型指针。

  • 数组指针和指针数组

    1. int (*ptr)[] 数组指针，指向整型数组的指针。
    2. int *ptr[] 指针数组，存放指针的数组

内存管理

  • 三种分配方式：

    1. 静态存储区：编译的时候就分配好了，整个运行期间都存在，如全局变量，static变量
    2. 栈上面创建。函数内的局部变量都在栈上创建，函数结束时自动被释放。
    3. 堆上分配，使用malloc和new分配，free和delete释放

  • 常见内存错误：

    1. 内存未分配成功，却使用。使用内存前，检查指针是否是NULL。
    2. 未初始化就是用。例如数组没有初始化就是用。
    3. 内存越界。
    4. 内存泄露。分配后没有释放。
    5. 释放了内存却继续使用。内存释放后，指针指向NULL。

  • free和delete后，指针地址不变，指针变成了野指针，这时候用null。

  • malloc 和new的区别
    对于非内部变量，malloc不能执行对象的构造函数，free也不能调用析构函数

函数的高级特性

  • 重载

    1. 函数参数（顺序，类型），返回值不同可以重载
    2. 必须是在一个作用于
    3. c++不能重载C的程序，需要加入extern “C”

  • 重写（覆盖）

    1. 分别为子类和基类
    2. 函数名子相同，参数相同
    3. 基类函数有virtural关键字

  • 隐藏

    1. 派生类和基类函数名相同，参数不同，无论有没有virtual，基类被隐藏
    2. 如果派生类和基类同名，参数相同，基类没有virtual，基类隐藏。（如有基类有virtual,是覆盖）

  • 运算符重载
    operator加上运算符来表示函数。运算符函数参数出现在左右两侧。

  • 虚函数

    1. 为多态而生
    2. 析构函数建议设置为虚函数
    3. 构造函数不能是虚函数
    4. 虚函数只有声明没有定义是纯虚函数
