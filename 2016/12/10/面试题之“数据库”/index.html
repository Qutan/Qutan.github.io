面试题之“数据库”

发表于 2016-12-10   |   分类于 面试题目   |  

三层模式两层映射

模式：数据库中全部数据集中全部数据的逻辑结果和特征描述
外模式：数据库中部分数据集中局部数据的逻辑结构和特征描述
内模式：数据库的存储方式和物理结构
外模式-模式映射，保证数据的逻辑完整性，多对一的映射。
模式-内模式映射，保证数据的物理完成整，一对一的映射。

逻辑模型：按照计算机的观点对数据建模。
概念模型：指从用户需求对数据和信息建模。

数据模型额组成因素

  1. 数据结构 2.数据操作 3. 完整性约束
  • 实体完整性：主键不为空，且不能重复
  • 参照完整性：没有外键，或者外键是是被参照的主键
  • 用户定义完整性：某一具体应用的数据满足语义要求：非空，唯一，或者布尔型

关系代数

  • 和：在A和B都会出现结果中
  • 差：在A中出现，在B中没出现的
  • 笛卡尔积：A中的每一个，和B中的每一个组合
  • 投影：某几列
  • 选择：选择某几行

组合运算

  • 交：A和B同时出现得
  • 除法：A/B,找到A和B相同的列，这几列和B相同的行，这几行和B不同列即为所求。
  • 连接：在笛卡尔积的基础上做选择
    • 等值连接，某两列相等的连接
    • 自然连接，重复的公共列相等，只保留一个

扩充操作：

  • 外连接：自然连接时保留本该舍弃的那部分，写成NULL
  • 左外连接：只保留左边应该保留的
  • 右外连接：只保留右边应该保留的
  • 半连接：自然连接后投影，保留选择的那一半

SQL语句

  1. 定义
    CREATE SCHEMA/TABLE/VIEW/INDEX
    DROP

  2. 查询
    SELECT [ALL|DISTINCT] <算术表达式> FROM <表名列表>
    [WHERE <条件表达式>]
    [GROUP BY <>] [Having 条件]
    [ORDER BY <>]
    其中的条件可以是AND,OR,IN<值列表>，BETWEEN<范围>，LIKE

  3. 增删改
    插入 INSERT INTO
    删除 DELETE FROM
    修改 UPDATE SET

  4. 控制
    授权 GRNT
    取消 REVOKE

范式

  1. 1NF:没有重复的列
  2. 2NF:实体完全依赖主属性
  3. 3NF:属性不依赖于其他非主属性，消除传递依赖
  4. BCNF:不存在关键字依赖关键字

事物

ACID：原子性，一致性，隔离性，持续性

视图和表的区别

视图是一个虚表，数据库中只有定义而没有数据。数据存放在原来的表中，基本表的数据发生变化，视图也发生了变化。

锁

死锁：某个事务需要的资源都等待另外一个解除，导致长时间的等待。
两种锁，使并发的事务对数据库的操作可以同步：

  1. X锁：排他锁
    T对R加X锁后，解除X前，其他事务不能加锁
  2. S锁：共享锁
    加S后，其他可以加S,但不能加X
